Personal Site Observability Platform

This site runs on a Raspberry Pi homelab that I put together. Every piece of the stack is containerized, monitored, and designed to stay up with as little intervention as possible.

Infrastructure - Everything runs in containers, so the whole environment is reproducible and each service is isolated from the rest. If one container goes down, it doesn't take the others down with it.

Security - Traffic reaches the server through a Cloudflare Tunnel, which means there are zero public-facing ports. That zero-trust model has blocked over a hundred attempted network intrusions so far.

Observability - Prometheus scrapes metrics from every service, Grafana turns those numbers into dashboards I can read, and Loki aggregates logs so I can search across containers in one place.

Alerting - Alertmanager and Blackbox Exporter probe the site from the inside and outside. If something goes down, I know about it within a minute, which is how uptime stays above 99.9% weekly.

Kubernetes Homelab Platform

Migrated my single-node platform to a highly available k3s cluster, improving fault tolerance and cutting service downtime by 60% compared to the previous deployment.

Automation - Automated Kubernetes deployments with ArgoCD using a GitOps workflow, which reduced manual rollouts and made infrastructure changes more reliable.

CI/CD - Built GitHub Actions pipelines to containerize and deploy my personal site and NIDS services directly to the cluster.

Next step - I'm planning to expand the platform with AWS integrations to keep improving reliability and add storage for long-term data retention.
